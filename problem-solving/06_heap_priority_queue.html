Pattern 06

Heap / Priority Queue

Use when you need repeated min/max access, Top-K elements, merging K sorted sequences, or a streaming median. Java's PriorityQueue is a min-heap; pass Comparator.reverseOrder() for max-heap.

Signal — "K-th largest/smallest", "top K", "merge K sorted"
Complexity — Insert O(log n) · Peek O(1) · Poll O(log n)
Two-heap trick — streaming median in O(log n)

Core Concepts

A binary heap is a complete binary tree stored as an array. For node at index i: parent = (i-1)/2, left child = 2i+1, right child = 2i+2. Min-heap invariant: every parent ≤ its children.

🏆

Top-K

Keep min-heap of size k. If new element > peek, poll then offer. O(n log k).

🔀

Merge K Sorted

Push each list head into min-heap. Poll smallest, push its next. O(n log k).

📊

Streaming Median

Max-heap for lower half, min-heap for upper half. Balance sizes. Median from tops.

Sift-Up (Insert)

Add element at the next leaf position. Compare with parent. If smaller (min-heap), swap. Repeat up to root. Max O(log n) swaps.

Sift-Down (Poll)

Swap root with last element. Remove last. Compare new root with smaller child. Swap if needed. Repeat downward. Max O(log n) swaps.

Java Templates

KthLargest.java — Fixed Window (min-heap size k) 
// KTH LARGEST — min-heap keeps the k largest seen so far
public int findKthLargest(int[] nums, int k) {
    PriorityQueue<Integer> minH = new PriorityQueue<>();
    for (int n : nums) {
        minH.offer(n);
        if (minH.size() > k) minH.poll();
    }
    return minH.peek();
}
// Time O(n log k)  Space O(k)
MergeKLists.java — Variable Window 
// MERGE K SORTED LISTS — min-heap on node value
public ListNode mergeKLists(ListNode[] lists) {
    PriorityQueue<ListNode> pq = new PriorityQueue<>(
        (a, b) -> Integer.compare(a.val, b.val)
    );
    for (ListNode h : lists) if (h != null) pq.offer(h);
    ListNode dummy = new ListNode(0), cur = dummy;
    while (!pq.isEmpty()) {
        cur.next = pq.poll();
        cur = cur.next;
        if (cur.next != null) pq.offer(cur.next);
    }
    return dummy.next;
}
// Time O(n log k)  Space O(k)  where n = total nodes, k = list count
MedianFinder.java — Two-Heap Trick (HARD) 
// FIND MEDIAN FROM DATA STREAM — two balanced heaps
class MedianFinder {
    // lo = max-heap  (lower half)
    PriorityQueue<Integer> lo = new PriorityQueue<>(Comparator.reverseOrder());
    // hi = min-heap  (upper half)
    PriorityQueue<Integer> hi = new PriorityQueue<>();

    public void addNum(int n) {
        lo.offer(n);             // push to lower half
        hi.offer(lo.poll());      // ensure max(lo) <= min(hi)
        // keep lo.size() >= hi.size()
        if (lo.size() < hi.size()) lo.offer(hi.poll());
    }

    public double findMedian() {
        return lo.size() > hi.size()
            ? lo.peek()
            : (lo.peek() + hi.peek()) / 2.0;
    }
}
// NOTE: Use Integer.compare(a, b) NOT (a - b) — avoids int overflow!
// Time addNum O(log n)  findMedian O(1)  Space O(n)

Staff Engineer Angles

"How do you compute a real-time median of an infinite data stream with O(k) memory — not O(n)?"
The two-heap approach is O(n) memory. For truly memory-bounded streaming:

t-digest — mergeable sketch that approximates quantiles with configurable accuracy. Merge-friendly; used in Prometheus and Elasticsearch.

Greenwald-Khanna (GK) sketch — O(1/ε · log(εn)) space; provable (ε, δ)-quantile guarantee. Ordered list of summary tuples.

Reservoir sampling — uniform random sample of size k from stream; empirical median from sample. Bias-free for uniform streams, skewed for heavy-tailed distributions.
"How do you merge K=10,000 sorted files, each with 1B records, at exabyte scale?"
A single in-memory min-heap of 10,000 cursors is feasible (tiny RAM). The bottleneck is I/O:

Multi-pass external merge sort — if each file fits on one machine, stream-merge in passes. Classic MapReduce merge phase.

Polyphase merge sort — minimises tape/disk seeks by distributing initial runs across more than 2 drives. Optimal for sequential storage media.

Parallel merge tree — binary tree of merge workers; each worker merges two streams and forwards. Pipeline depth = log₂(K). Scales horizontally with network bandwidth as the new constraint.
