Domain Events and Event Storming

Chapter 3 - Domain events, event flow, storming, pub/sub, and eventual consistency.

3.1 -What is a Domain Event?Immutable fact that happened

Events capture past business truth and should never be mutated.

3.2 -OrderPlaced Event FlowFan-out to multiple handlers

One event drives independent workflows in billing, shipping, and notification contexts.

3.3 -Event Storming IntroductionTimeline with commands and events

Event storming reveals process flow and hidden domain decisions quickly.

3.4 -Aggregates and Event GenerationState transitions emit events

Aggregate state changes should produce explicit events for history and integration.

3.5 -Publish/Subscribe PatternPublisher with independent subscribers

Publish once, react many times. Subscribers stay decoupled from publisher internals.

3.6 -Eventual ConsistencyConsistency window between T1 and T2

Distributed contexts may be briefly out of sync until events are consumed.
