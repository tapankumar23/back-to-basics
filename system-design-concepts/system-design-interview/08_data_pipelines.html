Data Pipelines

Kafka-style queues, metrics/monitoring pipelines, and event aggregation

8.1 — Distributed Message Queue (Kafka-like) Partitions, consumer group offsets, leader election
Kafka partitions allow parallel producers/consumers. Each consumer group maintains its own offset — different groups can replay or lag independently. Leaders handle reads/writes; followers provide durability. On leader failure, a follower is elected via controller coordination.
8.2 — Metrics Monitoring Collection agents, TSDB, rollup aggregation, alerting
Storing raw metrics at 1s resolution for 30 days is expensive. Rollup aggregation compresses old data (1min averages, then 1hr averages) trading resolution for storage efficiency. Alert thresholds trigger notifications when a metric exceeds a configured ceiling.
8.3 — Ad Click Event Aggregation Lambda vs Kappa architecture, deduplication
Lambda architecture runs batch + streaming paths in parallel — batch for accuracy, streaming for freshness. Kappa simplifies to streaming only, using Kafka log replay for historical processing. Deduplication via fingerprint hashing drops duplicate click events before aggregation.
