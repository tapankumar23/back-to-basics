PATTERN 01

Sliding Window

Contiguous subarray/substring satisfying a constraint — O(n) by maintaining a window that expands right and contracts left.

Core Concept

Two variants share the same pointer-pair skeleton but differ in how the window contracts.

Fixed Window (size k)

Advance left by 1 every time right advances past k.

for i = k → n:
  add arr[i]
  remove arr[i-k]
  update answer
Variable Window

Shrink left while constraint violated, expand right freely.

for right = 0 → n:
  expand(arr[right])
  while violated:
    shrink(arr[left++])
  update answer

Algorithm Stepper

Array size: 10
Speed: ⚡
Step 0 / 0
left:0 right:-1 windowSum:0 answer:-∞
Invariant: window [left..right] satisfies constraint at all times

Java Code Template

// ===== FIXED WINDOW =====
public int maxSumFixed(int[] arr, int k) {
    int windowSum = 0, maxSum = Integer.MIN_VALUE;
    for (int i = 0; i < k; i++) windowSum += arr[i];
    maxSum = windowSum;
    for (int i = k; i < arr.length; i++) {
        windowSum += arr[i] - arr[i - k];
        maxSum = Math.max(maxSum, windowSum);
    }
    return maxSum;
}

// ===== VARIABLE WINDOW (general template) =====
public int variableWindow(int[] arr) {
    Map<Integer, Integer> freq = new HashMap<>();
    int left = 0, result = 0;
    for (int right = 0; right < arr.length; right++) {
        // EXPAND: add arr[right] to window state
        freq.merge(arr[right], 1, Integer::sum);
        // SHRINK: while constraint violated
        while (freq.size() > 2) {  // e.g., >k distinct
            freq.merge(arr[left], -1, Integer::sum);
            if (freq.get(arr[left]) == 0) freq.remove(arr[left]);
            left++;
        }
        result = Math.max(result, right - left + 1);
    }
    return result;
}

// ===== SLIDING WINDOW MAXIMUM (monotonic deque) =====
public int[] maxSlidingWindow(int[] nums, int k) {
    int n = nums.length;
    int[] result = new int[n - k + 1];
    ArrayDeque<Integer> dq = new ArrayDeque<>(); // stores indices
    for (int i = 0; i < n; i++) {
        while (!dq.isEmpty() && dq.peekFirst() < i - k + 1) dq.pollFirst();
        while (!dq.isEmpty() && nums[dq.peekLast()] < nums[i]) dq.pollLast();
        dq.offerLast(i);
        if (i >= k - 1) result[i - k + 1] = nums[dq.peekFirst()];
    }
    return result;
}

Staff Angles

Stream Context

Count distinct users in any 24-hour sliding window over an infinite event stream that won't fit in memory — how?

→ Time-bucketed windows (e.g., 1-hour buckets, keep 24), approximate distinct count with HyperLogLog per bucket (merge with union), or Bloom filters for membership. Redis TimeSeries with EXPIRE handles eviction. For exact counts, event-sourcing with a sorted-set keyed by user+timestamp.

Latency Context

Sliding window median is O(n log k). Feed-ranking service needs sub-10ms p99 at 1M RPS — what do you precompute or approximate?

→ Offline sorted structure (two heaps offline), t-digest for streaming quantiles with tunable accuracy, reservoir sampling for approximate median. At 1M RPS precompute percentiles in a separate aggregation tier and serve cached values with TTL of 100–500ms. Use LMAX Disruptor or ring buffers to avoid GC pressure.
