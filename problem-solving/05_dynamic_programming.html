Pattern 05

Dynamic Programming

Use when the problem has optimal substructure and overlapping subproblems. Identify state variables, write the recurrence, set base cases, then choose traversal order.

Signal — "optimal", "count ways", "min/max"
State — variables that uniquely identify a subproblem
Recurrence — how dp[i] depends on smaller states

Core Concepts

DP is about recognizing that the brute force recursion visits the same subproblems repeatedly. Memoize top-down or fill a table bottom-up. The table dimensions equal the number of state variables.

🎒

Knapsack

Include/exclude item. State: (item index, remaining capacity).

🧬

LCS / Strings

Two sequences. State: (i, j) positions. Match or skip.

📈

LIS Pattern

Subsequence with constraint. dp[i] = best ending at i.

🔲

Grid DP

Navigate grid. State: (row, col). Direction constraints.

Top-Down vs Bottom-Up

Top-down (memoization): write the recursion naturally, add a cache. Easier to reason about. Bottom-up (tabulation): fill table in dependency order. No stack overhead, easier to optimize space.

Space Optimization

If dp[i][j] only depends on dp[i-1][...], use two rows (rolling array). Reduces O(m*n) to O(n). For some problems, even O(1) extra space is achievable.

// 0/1 KNAPSACK — O(n*cap) time, O(n*cap) space
public int knapsack(int[] w, int[] v, int cap) {
    int n = w.length; int[][] dp = new int[n+1][cap+1];
    for (int i = 1; i <= n; i++)
        for (int c = 0; c <= cap; c++) {
            dp[i][c] = dp[i-1][c];
            if (w[i-1] <= c) dp[i][c] = Math.max(dp[i][c], dp[i-1][c-w[i-1]] + v[i-1]);
        }
    return dp[n][cap];
}

// LCS — O(mn) time and space
public int lcs(String s1, String s2) {
    int m = s1.length(), n = s2.length(); int[][] dp = new int[m+1][n+1];
    for (int i = 1; i <= m; i++)
        for (int j = 1; j <= n; j++)
            dp[i][j] = s1.charAt(i-1) == s2.charAt(j-1)
                ? dp[i-1][j-1] + 1
                : Math.max(dp[i-1][j], dp[i][j-1]);
    return dp[m][n];
}

// LIS — O(n log n) patience sorting
public int lis(int[] nums) {
    List<Integer> tails = new ArrayList<>();
    for (int num : nums) {
        int pos = Collections.binarySearch(tails, num);
        if (pos < 0) pos = -(pos + 1);
        if (pos == tails.size()) tails.add(num); else tails.set(pos, num);
    }
    return tails.size();
}

// COIN CHANGE (min coins) — O(n*amount)
public int coinChange(int[] coins, int amount) {
    int[] dp = new int[amount+1];
    Arrays.fill(dp, amount+1); dp[0] = 0;
    for (int i = 1; i <= amount; i++)
        for (int c : coins)
            if (c <= i) dp[i] = Math.min(dp[i], dp[i-c] + 1);
    return dp[amount] > amount ? -1 : dp[amount];
}

LIS Pattern
23. LIS (LC 300)
O(n log n) with patience sorting. Binary search on tails array.

Staff Engineer Angles

"DP table is O(m*n) = 10GB for m=n=100k. Streaming alignment where you only need the score?"
Use a rolling array: only keep two rows (or one row for some formulations), reducing space from O(m*n) to O(min(m,n)). For traceback, use Hirschberg's algorithm: divide-and-conquer recursion that computes the optimal alignment in O(n) space by finding the midpoint of the optimal path, then recursing. Time stays O(mn), space drops to O(n). For streaming where you can't store even one row, approximate alignment with MinHash or locality-sensitive hashing.
"Knapsack over 1B items that won't fit on one machine. How to partition and solve?"
Sort items by weight-to-value ratio and group them into buckets. Use branch-and-bound distributed across machines: each machine explores a subtree of the decision tree, pruning with LP relaxation upper bounds. For approximate solutions, FPTAS gives (1+epsilon) approximation in polynomial time by scaling and rounding item values. For exact large instances: split-in-half meet-in-the-middle (O(2^(n/2))) if n is small enough, or DP on deduplicated/grouped items when many items have identical properties.
