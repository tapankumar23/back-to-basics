Video & Object Storage

Video upload/transcoding pipelines, CDN streaming, and S3-style object storage

6.1 — YouTube: Video Upload Pipeline DAG-based parallel transcoding and distribution
The DAG task graph allows parallel transcoding — all quality levels are generated simultaneously. Task scheduler ensures each worker handles exactly one task and failures trigger retask. Splitting encoding from storage means the upload service stays fast while background workers handle the heavy lifting.
6.2 — YouTube: Video Streaming CDN edge caching and adaptive bitrate streaming
CDN serves video chunks from edge nodes near users, cutting latency dramatically. Adaptive bitrate streaming (ABR) adjusts quality every few seconds based on available bandwidth — critical for mobile users on variable connections. A CDN HIT returns data from the nearest edge; a MISS fetches from origin and caches for subsequent viewers.
6.3 — S3-like Object Storage Chunk distribution, erasure coding, and versioning
Erasure coding (like RAID-6) stores 6 data + 3 parity chunks. Any 6 of the 9 can reconstruct the file — tolerating 3 simultaneous node failures at lower storage overhead than full 3x replication. Metadata (bucket, key, version, chunk locations) lives in a fast key-value store separate from the actual data.
