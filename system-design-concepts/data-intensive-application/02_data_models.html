Cypher vs SPARQL — Same Query, Two Languages

      

        

          
Cypher (Neo4j)

          
MATCH (person:Person)
  -[:BORN_IN]->()-[:WITHIN*0..]->(us:Location)
WHERE us.name = "United States"
RETURN person.name

        

        

          
SPARQL (W3C standard)

          
PREFIX : <urn:example:>
SELECT ?personName WHERE {
  ?person :bornIn ?place .
  ?place  :within* :USA .
  ?person :name ?personName .
}

        

      

    

  


  
  

    

      
Section 6

      
Declarative vs Imperative Query Languages

      
SQL hides the how. MapReduce exposes it. The declarative/imperative divide runs through all of computing.

    


    

      

        
SQL — Declarative

        
SELECT family, count(*) AS cnt
FROM   animals
WHERE  family = 'Sharks'
GROUP BY family;

        

          
All Rows

          →
          
Filter Sharks

          →
          
Aggregate

          →

MapReduce — Imperative

        
db.animals.mapReduce(
  function map() {
    if (this.family === 'Sharks')
      emit(this.family, 1);
  },
  function reduce(key, vals) {
    return Array.sum(vals);
  }
);

        
        

          
        

      

    

  


  
  

    

      
Section 7

      
Schema-on-write vs Schema-on-read

      
Relational databases enforce schema at write time. Document databases check at read time. Each has costs and benefits.

    


    

      
    


    

      

        

          
Schema-on-write (Relational)

          
GATE

          

            
✗ Invalid doc bounces back

            
✓ Valid doc passes gate

            
⚙ ALTER TABLE — add column (migration required, may lock table)

          

        

      

      

        

          
Schema-on-read (Document)

          

            
✓ All docs accepted at write time

            
Read time — branch on field:

            

              
if (doc.phone != null) {
  // new code path
  usePhone(doc.phone);
} else {
  // old code path
  useEmail(doc.email);
}

            

          

        

      

    

  


  
  

    

      
Section 8

      
Datalog — Rules & Derived Facts

      
Datalog is the ancestor of SPARQL. Facts + rules = derived knowledge, computed step by step.

    


    

      

        
Facts, Rules, Derivations

        
Base facts:

        
within(Idaho, USA).
within(USA, NorthAmerica).
born_in(Alice, Idaho).

        
Rule (recursive):

        
within_r(X,Y) :- within(X,Y).
within_r(X,Z) :- within(X,Y), within_r(Y,Z).
lives_in(P,Z) :- born_in(P,X), within_r(X,Z).

Three-Way Query Comparison

        

          

            
Cypher

            
MATCH (p:Person)
  -[:BORN_IN]->()
  -[:WITHIN*0..]->(na:Location)
WHERE na.name = "NorthAmerica"
RETURN p.name

          

          

            
SPARQL

            
SELECT ?name WHERE {
  ?p :bornIn ?place .
  ?place :within* :NorthAmerica .
  ?p :name ?name .
}

          

          

            
Datalog

            
in_na(X) :- within_r(X, NorthAmerica).
answer(N) :- born_in(P,X), in_na(X),
             name(P, N).
