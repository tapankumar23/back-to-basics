Trees + BST

Hierarchical recursive divide-and-conquer. BST: left < root < right. Master DFS (pre/in/post-order), BFS (level-order), and reconstruction from traversals.

Core Concept — BST Structure

BST Property
For every node: all values in left subtree < node.val < all values in right subtree. Inorder traversal yields sorted sequence.
DFS Traversals
Pre-order: root → left → right (copy/serialize). In-order: sorted output. Post-order: delete/evaluate (children first).
BFS / Level-Order
Queue-based. Process level by level. Use queue size snapshot to group by level (LC 102).
BST [8, 3, 10, 1, 6, 14, 4, 7, 13] — D3 Layout
● Normal node ● Root Inorder: 1, 3, 4, 6, 7, 8, 10, 13, 14 (sorted)

Java Implementation

// TreeNode definition
class TreeNode {
    int val; TreeNode left, right;
    TreeNode(int v) { val = v; }
}

// ── Inorder Iterative (explicit stack) ──────────────────────────────────────
List<Integer> inorderIter(TreeNode root) {
    List<Integer> res = new ArrayList<>();
    ArrayDeque<TreeNode> s = new ArrayDeque<>();
    TreeNode c = root;
    while (c != null || !s.isEmpty()) {
        while (c != null) { s.push(c); c = c.left; }  // go left
        c = s.pop(); res.add(c.val);                  // visit
        c = c.right;                                      // go right
    }
    return res;
}

// ── Lowest Common Ancestor (LC 236) ─────────────────────────────────────────
TreeNode lca(TreeNode r, TreeNode p, TreeNode q) {
    if (r == null || r == p || r == q) return r;
    TreeNode l  = lca(r.left,  p, q);
    TreeNode ri = lca(r.right, p, q);
    if (l != null && ri != null) return r;  // found in both subtrees
    return l != null ? l : ri;
}

// ── Validate BST (LC 98) — pass min/max bounds, NOT just child comparison ───
boolean isValidBST(TreeNode r, long min, long max) {
    if (r == null) return true;
    if (r.val <= min || r.val >= max) return false;
    return isValidBST(r.left,  min,    r.val)
        && isValidBST(r.right, r.val,  max);
}
// Call: isValidBST(root, Long.MIN_VALUE, Long.MAX_VALUE)

// ── Diameter of Binary Tree (LC 543) ────────────────────────────────────────
int maxDiam = 0;
int diameter(TreeNode node) {
    if (node == null) return 0;
    int L = diameter(node.left);
    int R = diameter(node.right);
    maxDiam = Math.max(maxDiam, L + R);  // path through this node
    return 1 + Math.max(L, R);            // height upward
}

// ── Binary Tree Max Path Sum (LC 124) ───────────────────────────────────────
int maxSum = Integer.MIN_VALUE;
int maxGain(TreeNode node) {
    if (node == null) return 0;
    int L = Math.max(0, maxGain(node.left));   // clamp negatives to 0
    int R = Math.max(0, maxGain(node.right));
    maxSum = Math.max(maxSum, L + node.val + R);
    return node.val + Math.max(L, R);  // only ONE branch can go up
}

// ── Kth Smallest in BST (LC 230) — inorder counter ──────────────────────────
int kthCount = 0, kthResult = -1;
void kthSmallest(TreeNode node, int k) {
    if (node == null || kthCount == k) return;
    kthSmallest(node.left, k);
    if (++kthCount == k) { kthResult = node.val; return; }
    kthSmallest(node.right, k);
}

// ── Serialize/Deserialize (LC 297) — preorder with null markers ─────────────
String serialize(TreeNode root) {
    if (root == null) return "#";
    return root.val + "," + serialize(root.left) + "," + serialize(root.right);
}
TreeNode deserialize(String data) {
    Queue<String> q = new LinkedList<>(Arrays.asList(data.split(",")));
    return buildTree(q);
}
TreeNode buildTree(Queue<String> q) {
    String val = q.poll();
    if ("#".equals(val)) return null;
    TreeNode node = new TreeNode(Integer.parseInt(val));
    node.left  = buildTree(q);
    node.right = buildTree(q);
    return node;
}

// ── Construct from Preorder + Inorder (LC 105) ──────────────────────────────
TreeNode buildFromTraversals(int[] pre, int[] in,
                               int pL, int pR, int iL, int iR,
                               Map<Integer,Integer> idx) {
    if (pL > pR) return null;
    int rootVal = pre[pL];
    TreeNode root = new TreeNode(rootVal);
    int m    = idx.get(rootVal);   // index in inorder
    int leftSz = m - iL;
    root.left  = buildFromTraversals(pre, in, pL+1,       pL+leftSz, iL,   m-1, idx);
    root.right = buildFromTraversals(pre, in, pL+leftSz+1, pR,        m+1, iR,   idx);
    return root;
}

// ── Delete Node in BST (LC 450) — 3 cases ───────────────────────────────────
TreeNode deleteNode(TreeNode root, int key) {
    if (root == null) return null;
    if      (key < root.val) root.left  = deleteNode(root.left,  key);
    else if (key > root.val) root.right = deleteNode(root.right, key);
    else {
        if (root.left  == null) return root.right;  // leaf or one child
        if (root.right == null) return root.left;
        TreeNode succ = root.right;                    // inorder successor
        while (succ.left != null) succ = succ.left;
        root.val   = succ.val;
        root.right = deleteNode(root.right, succ.val);
    }
    return root;
}

Staff-Level Angles

Q: BST degenerates to O(n) under sequential inserts — production fix?

Inserting 1, 2, 3, 4… into a plain BST creates a linked list: every lookup is O(n). You need a self-balancing BST.

AVL trees maintain |height(left) − height(right)| ≤ 1 via single/double rotations — guaranteed O(log n).
Red-Black trees relax balance slightly (fewer rotations) — O(log n) amortised; Java's TreeMap / TreeSet are Red-Black.
B+ trees minimise disk I/O (n-ary nodes); every RDBMS index is a B+ tree.
In prod: never implement your own — use TreeMap for in-memory sorted maps or a DB index for disk. If you need concurrent access, ConcurrentSkipListMap (lock-free O(log n)) often beats synchronized Red-Black in Java.

Q: LCA queries on a 1B-node tree not fitting in memory — O(log n) without full load?

A single DFS for each query is O(n) and requires the whole tree in RAM. Three production approaches:

1. Binary Lifting / Sparse Tables — precompute ancestor[node][2^k] for each node. LCA in O(log n) per query, O(n log n) preprocessing. Requires O(n log n) space but fits on disk with on-demand page loading.
2. Euler Tour + Range Minimum Query — DFS creates an Euler tour of length 2n−1; LCA(u,v) = minimum-depth node in the tour between u and v. RMQ answers in O(1) after O(n) preprocessing with sparse table. Elegant and cache-friendly.
3. Batch DFS with precomputed DFS-in/out numbering — assign DFS timestamps; ancestor check becomes a range query. Batch 1000s of LCA queries per disk pass, amortising I/O. Used in Google's Dremel / Protobufs nested repetition levels.
