What Is Here

A compact map of the full interview toolkit.

The content moves from core patterns to higher-composition topics. Early modules teach recognition; later ones combine multiple ideas in the same problem and force better state modeling.

Template-first learning

Each module anchors on the reusable skeleton: where pointers move, when a stack pops, what state a DP cell means, or how a graph traversal marks progress.

Visual intuition

The interactive sections are not decorative. They make transitions, window boundaries, event ordering, and graph frontier expansion easier to reason about during interviews.

Interview transfer

The goal is to recognize a pattern quickly, explain why it fits, and implement it in Java without rebuilding the idea from scratch every time.

Learning Roadmap

Four clusters that build on each other.

If you are revising, jump by cluster. If you are rebuilding fundamentals, go in order and treat each section as a pattern family rather than a one-off page.
