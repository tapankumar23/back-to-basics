Pattern 03

Recursion

Use when the problem has self-similar substructure — a base case that stops the recursion and a recursive case that reduces the problem. If subproblems overlap, memoize.

Linear — one recursive call
Branching — exponential call tree
Tail — convertible to loop
Time O(branches^depth)

Core Concepts

Every recursive function needs two parts: a base case (stop condition) and a recursive case (reduce and call). Three structural types determine complexity.

➡️

Linear Recursion

One recursive call per frame. O(n) depth. Reversing a linked list, factorial, power.

🌳

Branching Recursion

Two or more calls per frame. Exponential: O(2^n) or O(n!). Subsets, permutations, parens.

🔄

Tail Recursion

Recursive call is the last operation. JVM cannot optimize, but logic converts to a loop.

The Recursion Contract

Assume the recursive call returns the correct answer for a smaller input. Your job: combine the result for the current input. Trust the contract.

Memoization Trigger

If you notice the same arguments appearing multiple times in the call tree, add a HashMap cache. Transforms O(2^n) to O(n).

// SUBSEQUENCES — include/exclude pattern, O(2^n)
void subsequences(int[] arr, int idx, List<Integer> cur, List<List<Integer>> res) {
    if (idx == arr.length) { res.add(new ArrayList<>(cur)); return; }
    cur.add(arr[idx]); // include
    subsequences(arr, idx+1, cur, res);
    cur.remove(cur.size()-1); // exclude (backtrack)
    subsequences(arr, idx+1, cur, res);
}

// PERMUTATIONS — swap-based, O(n*n!)
void permute(int[] arr, int start, List<String> res) {
    if (start == arr.length) { res.add(Arrays.toString(arr)); return; }
    for (int i = start; i < arr.length; i++) {
        swap(arr, start, i);
        permute(arr, start+1, res);
        swap(arr, start, i); // restore
    }
}

// BALANCED PARENTHESES — open >= close invariant
void genParens(int open, int close, int n, StringBuilder sb, List<String> res) {
    if (sb.length() == 2*n) { res.add(sb.toString()); return; }
    if (open < n)     { sb.append('('); genParens(open+1, close, n, sb, res); sb.deleteCharAt(sb.length()-1); }
    if (close < open) { sb.append(')'); genParens(open, close+1, n, sb, res); sb.deleteCharAt(sb.length()-1); }
}

Problem Set

12 recursion problems ordered by concept. Study each invariant.

1. Print All Subsequences
Branching factor 2 (include/exclude). Every subset of the array.
O(2^n)
2. Generate All Permutations
Swap-based. Fixes element at start, recurses on rest.
O(n*n!)
3. Letter Case Permutation (LC 784)
Branch at each letter: lower AND upper case. Skip digits.
O(2^letters)
4. Balanced Parentheses (LC 22)
Invariant: open >= close always. Prune invalid branches early.
O(4^n / sqrt(n))
5. N-bit Binary: More 1s than 0s
Prune branch when zeros > ones so far. Constraint recursion.
O(2^n) pruned
6. Tower of Hanoi
Move n-1 to aux, move largest to dest, move n-1 from aux to dest.
O(2^n - 1)
7. Josephus Problem
f(n,k) = (f(n-1,k) + k) % n. Offset mapping back to original indices.
O(n)
8. Sort Array Using Recursion
Demonstrates loop to recursion equivalence. Insert last element into sorted subarray.
O(n^2)
9. Delete Middle of Stack
Recursion as implicit extra stack. Pop, recurse, push back.
O(n)
10. Reverse a Stack
Two helpers: insertAtBottom + reverse. No extra data structure.
O(n^2)
11. K-th Symbol in Grammar (LC 779)
Row n = row(n-1) + complement(row(n-1)). Navigate to parent.
O(n)
12. Power Set (LC 78)
2^n subsets via include/exclude. Classic branching recursion.
O(n*2^n)

Staff Engineer Angles

"Generate only the k-th lexicographic permutation of 1M elements without materializing all permutations."
Use Lehmer codes and the factorial number system. For n elements, the k-th permutation's first digit is k / (n-1)! — no recursion tree needed. O(n^2) with a sorted list, O(n log n) with a Fenwick tree for position queries. Never generate more than O(n) elements.
"Deep recursive DFS on 500k-node graph hits StackOverflowError on JVM. Production fix?"
Convert to iterative DFS with an explicit ArrayDeque stack. Alternatively, use the trampoline pattern: return a thunk (lambda) instead of recursing, then a loop evaluates thunks. For JVM specifically: increase -Xss stack size (band-aid), or redesign with Continuation/Virtual Threads (Project Loom). ArrayDeque approach is the production standard.
