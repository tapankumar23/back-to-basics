Hashing · Intervals & Sweep Line

HashMap gives O(1) lookup and frequency counting. Intervals model time-ranges, booking windows, and coverage. Sweep line converts 2-D overlap problems into a 1-D sorted scan — counting "how many things are active right now" becomes trivially O(n log n).

Core Concept

Hashing

Use a HashMap when you need O(1) "have I seen this before?" or frequency counting. Canonical use-cases: anagram detection (sorted-char key), two-sum complement lookup, longest consecutive run (HashSet boundary check).

Intervals

Sort intervals by start time. Merging overlaps: if next.start ≤ current.end, extend. Checking overlap: [a,b] and [c,d] overlap iff a < d && c < b. Keep earliest-end to greedily remove minimum intervals.

Sweep Line

Explode each interval into two events: (start, +1) and (end, -1). Sort all events. Scan left to right maintaining a running counter. Peak counter = minimum rooms / maximum concurrent meetings.

Java Code

// LC 49 — Group Anagrams
// Sort each word to get a canonical key, then group into map.
// Time O(n * k log k)   Space O(n*k)   k = max word length

class Solution {
    public List<List<String>> groupAnagrams(String[] strs) {
        Map<String, List<String>> map = new HashMap<>();

        for (String s : strs) {
            char[] chars = s.toCharArray();
            Arrays.sort(chars);
            String key = new String(chars);    // sorted chars = canonical key
            map.computeIfAbsent(key, k -> new ArrayList<>()).add(s);
        }

        return new ArrayList<>(map.values());
    }
}

// Alternative: count-array key avoids sorting -> O(n*k) total
private String countKey(String s) {
    int[] cnt = new int[26];
    for (char c : s.toCharArray()) cnt[c - 'a']++;
    return Arrays.toString(cnt);
}
// LC 253 — Meeting Rooms II (min-heap approach)
// Sort by start. Min-heap tracks end-times of ongoing meetings.
// If heap.peek() <= start, reuse that room (poll + offer new end).
// Otherwise open a new room. Heap size = rooms needed at the end.
// Time O(n log n)   Space O(n)

class Solution {
    public int minMeetingRooms(int[][] intervals) {
        if (intervals == null || intervals.length == 0) return 0;

        Arrays.sort(intervals, (a, b) -> a[0] - b[0]); // sort by start
        PriorityQueue<Integer> heap = new PriorityQueue<>(); // min-heap of end times

        for (int[] meeting : intervals) {
            int start = meeting[0], end = meeting[1];
            if (!heap.isEmpty() && heap.peek() <= start) {
                heap.poll();    // free the earliest-ending room
            }
            heap.offer(end); // book this meeting into a room
        }

        return heap.size(); // rooms still in use = min rooms needed
    }
}

// Sweep-line equivalent — same result, cleaner mental model:
public int minMeetingRoomsSweep(int[][] intervals) {
    int n = intervals.length;
    int[] starts = new int[n];
    int[] ends   = new int[n];
    for (int i = 0; i < n; i++) {
        starts[i] = intervals[i][0];
        ends[i]   = intervals[i][1];
    }
    Arrays.sort(starts);
    Arrays.sort(ends);

    int rooms = 0, endPtr = 0;
    for (int i = 0; i < n; i++) {
        if (starts[i] < ends[endPtr]) rooms++;
        else endPtr++;
    }
    return rooms;
}
// LC 128 — Longest Consecutive Sequence
// Key insight: only start counting from a sequence boundary.
// n is a boundary start iff (n-1) is NOT in the set.
// This prevents O(n^2) re-counting of sub-sequences.
// Time O(n)   Space O(n)

class Solution {
    public int longestConsecutive(int[] nums) {
        Set<Integer> set = new HashSet<>();
        for (int n : nums) set.add(n);

        int best = 0;

        for (int n : set) {
            if (!set.contains(n - 1)) {         // sequence start boundary
                int curr = n, len = 1;
                while (set.contains(curr + 1)) {
                    curr++;
                    len++;
                }
                best = Math.max(best, len);
            }
        }
        return best;
    }
}

// Trace on [100,4,200,1,3,2]:
// set = {1,2,3,4,100,200}
// n=1:   (0 not in set)   walk 1,2,3,4  len=4  <-- answer
// n=100: (99 not in set)  len=1
// n=200: (199 not in set) len=1
// result = 4

Problem Set

  • LC 49
    Group Anagrams HashMap
    Sort each string to produce a canonical key; group words sharing the same key into one bucket.
  • LC 692
    Top K Frequent Words HashMap + PQ
    Build frequency map; push into min-heap of size k. Tie-break by reverse-lexicographic order in comparator.
  • LC 128
    Longest Consecutive Sequence HashSet
    Insert all into a HashSet; only begin counting when (num-1) is absent — that marks the true sequence start.
  • LC 454
    Four Sum Count HashMap
    Store all (A+B) sums in a map; for each (C+D) pair count how many -(C+D) matches exist in the map.
  • LC 252
    Meeting Rooms I Sort
    Sort by start time; return false if any adjacent pair overlaps: intervals[i].end > intervals[i+1].start.
  • LC 253
    Meeting Rooms II Sweep
    Sweep line or min-heap of end times; heap size at peak equals the minimum rooms required.
  • LC 435
    Non-Overlapping Intervals Greedy
    Sort by end time; greedily keep each interval whose start is at or after the last kept end. Count removed = total minus kept.
  • LC 986
    Interval List Intersections Two Pointers
    Two pointers on both sorted lists; intersection = [max(s1,s2), min(e1,e2)]. Advance the pointer with the smaller end time.
  • LC 729/732
    My Calendar I & III TreeMap
    Use a TreeMap as a difference array: +1 at start, -1 at end. A prefix scan gives the active count; detect conflicts or the global peak.
  • LC 218
    Skyline Problem Hard Sweep
    Sweep line with max-heap or TreeMap: add building height on left edge, remove on right edge. Emit a key point whenever the max height changes.
  • EPI
    Employee Free Time Merge Intervals
    Flatten all employees' intervals into one list, sort and merge overlapping blocks, then collect the gaps between merged blocks.

Staff Angles

Find max concurrent meetings globally across 10 billion events on 1000 shards — without pulling all data to one node?
Partition events by time bucket (e.g., one bucket per hour). Within each shard, run the local sweep line to produce a compact per-bucket active-count series — not raw events. Ship only those arrays (or per-bucket peaks) to an aggregator tier. The aggregator does a hierarchical merge: align timestamps, sum per-bucket local counts across shards, then scan for the global peak. Network transfer drops from O(10B events) to O(shards x buckets). Edge case: an interval that spans bucket boundaries must be propagated — track which intervals are still open at the end of each bucket and inject synthetic start events in the next bucket on the destination shard.
HashMap frequency counter on a 10 Gbps packet stream — what is the failure mode and how do you mitigate?
A naive HashMap runs out of memory: 10 Gbps carries 10-50 million distinct flows per second; keeping every key blows the heap and triggers GC storms under high throughput. Mitigations: (1) Count-Min Sketch — a probabilistic 2-D array with d hash functions; space O(1 / epsilon * log(1 / delta)); frequency estimates are always over-counts by at most epsilon x N with probability 1 - delta. (2) Heavy Hitters / Space Saving algorithm — exactly tracks top-k flows guaranteed to appear more than N/k times; O(k) space. (3) Flow table with LRU eviction — bound table size, evict least-recently-seen flows and accumulate their counts into a background bucket. For monitoring dashboards, Count-Min Sketch is the standard choice. For billing or security anomaly detection, Space Saving gives stronger guarantees.
