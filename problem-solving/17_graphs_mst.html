PATTERN 17

Graphs: MST

Find the minimum-cost spanning tree with Kruskal (DSU-based) or Prim (heap-based)

Core Concept

What is an MST?

A spanning tree that connects all V vertices using exactly V−1 edges with minimum total edge weight. No cycles, fully connected.

Key properties: unique if all edge weights are distinct; cut property guarantees the minimum-weight edge crossing any cut is in every MST.

Kruskal's Algorithm

Sort all edges by weight; greedily add an edge if it doesn't form a cycle — use DSU (Union-Find) to detect cycles in near-O(1).

Time: O(E log E) dominated by sort; DSU ops are effectively O(E α(V)).

Sort step: Collect all edges and sort ascending by weight.
Example: [(0,1,2),(1,2,3),(1,4,5),(0,3,6),(2,4,7),(1,3,8),(3,4,9)]

Prim's Algorithm

Grow MST from a start node; maintain a min-heap of (weight, vertex) for the "cut" edges between MST and non-MST vertices.

Time: O(E log V) with binary heap — better on dense graphs where E ≈ V².

Java Code Template

// ===== KRUSKAL'S MST (DSU-based) =====
public int kruskalMST(int n, int[][] edges) {
    // edges[i] = {u, v, weight}
    Arrays.sort(edges, (a, b) -> a[2] - b[2]);
    int[] parent = new int[n], rank = new int[n];
    for (int i = 0; i < n; i++) parent[i] = i;
    int totalCost = 0, edgesAdded = 0;
    for (int[] edge : edges) {
        int u = edge[0], v = edge[1], w = edge[2];
        int pu = find(parent, u), pv = find(parent, v);
        if (pu != pv) {
            union(parent, rank, pu, pv);
            totalCost += w;
            if (++edgesAdded == n - 1) break;
        }
    }
    return edgesAdded == n - 1 ? totalCost : -1; // -1 if disconnected
}

private int find(int[] parent, int x) {
    if (parent[x] != x) parent[x] = find(parent, parent[x]);
    return parent[x];
}

private void union(int[] parent, int[] rank, int a, int b) {
    if (rank[a] < rank[b]) { int t = a; a = b; b = t; }
    parent[b] = a;
    if (rank[a] == rank[b]) rank[a]++;
}

// ===== PRIM'S MST (Min-Heap) =====
public int primMST(int n, List<List<int[]>> adj) {
    // adj.get(u) = list of {v, weight}
    boolean[] inMST = new boolean[n];
    PriorityQueue<int[]> pq = new PriorityQueue<>((a,b) -> Integer.compare(a[0],b[0]));
    pq.offer(new int[]{0, 0}); // {weight, node}
    int totalCost = 0;
    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int w = cur[0], u = cur[1];
        if (inMST[u]) continue;
        inMST[u] = true;
        totalCost += w;
        for (int[] nei : adj.get(u)) {
            if (!inMST[nei[0]]) pq.offer(new int[]{nei[1], nei[0]});
        }
    }
    return totalCost;
}

Staff-Level Prompts

Distributed MST

A graph has 10 billion edges partitioned across 100 machines. How do you compute a global MST? The graph is too large to centralize.

→ Borůvka's algorithm is partition-friendly: in each round, every component independently finds its minimum-weight outgoing edge (parallelizable). Merge those edges, contract the graph, repeat for O(log V) rounds. Communication cost per round: each machine sends O(V_local) edges. Total: O(E/machines · log V) work, O(V · log V) messages across rounds. Compare to GHS (Gallager-Humblet-Spira) for asynchronous distributed MST — O(E + V log V) messages but complex leader-election per phase. At Google/Meta scale: use a MapReduce pipeline where Map emits (component_id, min_outgoing_edge), Reduce merges with a centralized DSU snapshot per round.

Streaming Edges

Edges arrive in a stream; you have only O(V) memory. Can you maintain an approximate or exact MST as edges arrive?

→ Exact O(V) maintenance is impossible in general (you may need to recall any past edge). Approximate approaches: L-sketches / graph sketches (Ahn, Guha, McGregor 2012) represent graph connectivity in O(V polylog V) bits using linear sketches — mergeable and streamable. For MST specifically, maintain a matroid intersection structure: a set of edges is MST-feasible iff it forms a forest (graphic matroid). The online greedy algorithm keeps the cheapest forest; when a new edge arrives, add it if it reduces total MST cost (replace the heaviest edge on the new edge's path if cheaper). This requires O(V) space with a link-cut tree for path-max queries in O(log V). Discuss when approximate answers are acceptable: for network provisioning, ±ε fraction of optimal MST cost is often fine — use random edge sampling (Karger's algorithm).
