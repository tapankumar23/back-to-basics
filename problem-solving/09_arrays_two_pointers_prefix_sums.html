Arrays, Two Pointers & Prefix Sums

Must-Know Problems

    Subarray Sum Equals K LC 560
    Prefix sum + HashMap: count(prefix[j] - k) gives subarrays ending at j with sum k.
  • 2
    Range Sum Query Immutable LC 303
    Classical 1D prefix sum. Build once, answer in O(1).
  • 3
    Range Sum Query 2D Immutable LC 304
    2D prefix: dp[i][j] = sum of top-left rectangle. Inclusion-exclusion for range query.
  • 4
    Product of Array Except Self LC 238
    left-prefix[i] * right-suffix[i]. No division. O(n) time O(1) extra.
  • 5
    Maximum Subarray LC 53
    Kadane's: cur = max(nums[i], cur + nums[i]). Track global max. O(n).
  • 6
    Minimum Size Subarray Sum LC 209
    Two-pointer variable window on positive array. Shrink from left when sum >= target.
  • 7
    Sort Colors LC 75
    Dutch National Flag: three-way partition with lo/mid/hi pointers. Single pass.
  • 8
    Trapping Rain Water LC 42
    Two-pointer with maxLeft and maxRight. Water at i = min(maxL, maxR) - height[i].
  • 9
    Squares of Sorted Array LC 977
    Two pointers from both ends, place larger square at back of result array.
  • 10
    3Sum Closest LC 16
    Sort + two-pointer. Track minimum |sum - target| across all windows.
  • 11
    Difference Array Range Update
    diff[l] += v; diff[r+1] -= v; then prefix sum rebuilds the final array in O(n).

Staff-Level Angles

    

      
Q: Range sum queries over a mutable 1B-element array with 10M updates/sec - better than prefix sum?

      
Fenwick tree (Binary Indexed Tree) gives O(log n) for both point update and prefix query - 10x smaller constant than segment tree due to cache-friendly layout. For range updates + range queries use a dual BIT. SIMD vectorization on prefix sum rebuild can hit 20GB/s on modern CPUs but can't beat BIT on mixed workloads. For approximate answers use sampling with Chebyshev bounds.

    

    

      
Q: 2D prefix sums for ML image processing kernels at 100MP images - limits and alternatives?

      
Summed area table (integral image) is the standard - used in Viola-Jones, SURF, and many CNN accelerators. At 100MP the table fits in ~800MB (float32). Cache-tiling is critical: scan in Z-order (Morton code) to improve L2 hit rate. On GPU, parallel prefix scan in CUDA achieves near-peak memory bandwidth. For repeated queries over the same image this dominates; for single-query per image skip the build and use SIMD sliding window sum instead.
