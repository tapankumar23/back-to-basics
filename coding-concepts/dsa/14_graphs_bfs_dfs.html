PATTERN 14

Graphs: BFS + DFS

BFS for shortest path in unweighted graphs, level-by-level. DFS for connected components, cycle detection, path existence. Both O(V+E).

When to Use

BFS — Shortest Path
Unweighted edges, level-by-level exploration. Each dequeued node has minimal hop distance from source. Use multi-source BFS for simultaneous starts.
DFS — Connectivity & Cycles
Connected components, cycle detection via 3-color (WHITE/GRAY/BLACK), topological sort, island counting. Recursive or explicit stack.
Both — O(V + E)
V vertices, E edges. Adjacency list preferred over matrix when sparse. BFS uses O(V) queue space; DFS uses O(V) stack/recursion depth.

// ─── BFS Template ───────────────────────────────────────────────────────────
void bfs(int start, List<List<Integer>> adj, int n) {
    boolean[] v = new boolean[n];
    Queue<Integer> q = new ArrayDeque<>();
    q.offer(start);
    v[start] = true;                     // mark BEFORE enqueue, not after dequeue
    while (!q.isEmpty()) {
        int u = q.poll();
        // process u here
        for (int nb : adj.get(u))
            if (!v[nb]) { v[nb] = true; q.offer(nb); }
    }
}

// ─── Iterative DFS ──────────────────────────────────────────────────────────
void dfsIter(int start, List<List<Integer>> adj, int n) {
    boolean[] v = new boolean[n];
    ArrayDeque<Integer> s = new ArrayDeque<>();
    s.push(start);
    while (!s.isEmpty()) {
        int u = s.pop();
        if (v[u]) continue;              // skip if already visited
        v[u] = true;
        for (int nb : adj.get(u))
            if (!v[nb]) s.push(nb);
    }
}

// ─── Number of Islands (LC 200) ─────────────────────────────────────────────
public int numIslands(char[][] g) {
    int c = 0;
    for (int r = 0; r < g.length; r++)
        for (int col = 0; col < g[0].length; col++)
            if (g[r][col] == '1') { sink(g, r, col); c++; }
    return c;
}
void sink(char[][] g, int r, int c) {
    if (r < 0 || r >= g.length || c < 0 || c >= g[0].length || g[r][c] != '1') return;
    g[r][c] = '0';                       // mark visited by flipping
    sink(g,r+1,c); sink(g,r-1,c); sink(g,r,c+1); sink(g,r,c-1);
}

// ─── Cycle Detection — 3-color DFS ──────────────────────────────────────────
// 0=WHITE(unvisited) 1=GRAY(in current path) 2=BLACK(done)
boolean hasCycle(int n, List<List<Integer>> adj) {
    int[] col = new int[n];
    for (int i = 0; i < n; i++)
        if (col[i] == 0 && dfc(i, adj, col)) return true;
    return false;
}
boolean dfc(int u, List<List<Integer>> adj, int[] col) {
    col[u] = 1;                           // GRAY — currently on path
    for (int v : adj.get(u)) {
        if (col[v] == 1) return true;    // back-edge → cycle
        if (col[v] == 0 && dfc(v, adj, col)) return true;
    }
    col[u] = 2;                           // BLACK — fully processed
    return false;
}

Staff / Senior Engineer Angles

BFS on social graph — 3B nodes, 300B edges. Find connected components?

Single-machine BFS is impossible at this scale (graph won't fit in RAM). You need distributed graph processing.

Pregel / Apache Spark GraphX: vertex-centric "think like a vertex" model. Each superstep: every vertex sends messages to neighbors, receives messages, updates state. Label Propagation Algorithm (LPA) runs iteratively — each vertex adopts the majority label of its neighbors. Converges in O(diameter) supersteps.

Partitioning: vertex-cut (edge goes to one machine, vertices replicated) beats edge-cut for power-law graphs (Facebook, Twitter) where high-degree hub replication in edge-cut causes enormous messages.

Optimizations: bitmap visited sets, delta-BFS (only active frontier propagated), async execution (Giraph+), SSD-backed graph stores (GraphChi) for graphs larger than RAM.

Graph edges arrive as a stream (add/remove). Maintain connected components online?

Fully-dynamic connectivity is one of the hardest problems in data structures.

Insertions only (link): Union-Find / DSU with path compression + union by rank gives near O(1) amortized per operation. Straightforward.

Deletions only (cut): Offline — process in reverse, convert cuts to links, run DSU. Online — Euler Tour Trees (ETTs) maintain spanning forest; O(log²n) per operation.

Fully dynamic (link + cut): Link-Cut Trees (Sleator-Tarjan) O(log n) per link/cut/connectivity query. HDT (Holm-de Lichtenberg-Thorup) randomized O(log²n) per update. In practice, HDT is used in systems needing exact online connectivity (e.g., network topology managers).

Approximate at scale: Streaming connected components via sketches (AGM'12) — O(n polylog n) space, O(1) per edge, single-pass. Used in massive streaming graph systems.
