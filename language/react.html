Concept Orbit

React is a component system around UI state and predictable re-renders.

Start with the mental model: React composes screens from reusable components. Each component receives props, owns or derives state, and reacts to user input by re-rendering the UI efficiently.

Hooks Pocket Guide

Hooks let function components manage behavior without class boilerplate.

The most important early question is not just what each Hook does, but why it exists and when it should not be used. Hooks should stay predictable, top-level, and focused.

useState: local state
useEffect: side effects
useRef: mutable holder
useMemo: cached computation
useCallback: stable function
useContext: shared value
When To Use
  • useState() for local UI changes
  • useEffect() for subscriptions, fetches, timers
  • useMemo() only for meaningful recalculation cost
When To Be Careful
  • Do not call Hooks conditionally
  • Avoid overusing memoization for tiny computations
  • Keep custom Hooks single-purpose and reusable
Render Flow

Every React update follows a learnable path from event to UI change.

A user event triggers state change, React schedules a render, compares the new tree with the previous one, and commits only the needed DOM mutations. This is the basis for efficient updates.

User Event Click, input, network response, or prop change triggers work.
State Update setState or parent props produce a new render path.
Diffing React compares previous and next virtual tree representations.
Commit Only the necessary DOM nodes are patched in the browser.
Architecture Notes

Senior-level React thinking is usually about boundaries, not syntax.

Strong React engineers design reusable components, separate UI from business logic, choose the right level of state sharing, and avoid unnecessary optimization until measurement shows a real issue.

Healthy Patterns
  • Prefer clear component boundaries
  • Use local state until sharing becomes necessary
  • Test behavior instead of internal implementation
Red Flags
  • Huge components with mixed concerns
  • Context used for every type of state
  • Memoization added everywhere without profiling

React Glossary

Dictionary of terms to understand in React

Use this tab as a quick-reference glossary. These are the core React terms someone should understand before moving into advanced patterns, architecture, and interview-style discussion.

Library vs Framework

React is generally called a library because it focuses on UI, while frameworks usually prescribe a broader application structure.

Component

A reusable building block that returns UI. Components help split large interfaces into smaller logical pieces.

Functional Component

A modern React component written as a JavaScript function, often paired with Hooks for state and side effects.

Class Component

An older React component style based on JavaScript classes and lifecycle methods such as componentDidMount().

JSX

A syntax extension that lets you describe UI in JavaScript using HTML-like markup.

Props

Inputs passed from a parent component to a child component. Props are read-only inside the receiving component.

State

Internal component data that can change over time and trigger a re-render when updated.

Re-render

The process where React runs a component again to produce an updated UI after props or state change.

Virtual DOM

An in-memory representation of the UI that React compares to detect what changed before touching the real DOM.

Hook

A special React function such as useState() or useEffect() that adds behavior to function components.

useState

A Hook used to store and update local component state.

useEffect

A Hook used for side effects such as API calls, timers, subscriptions, or syncing with external systems.

useLayoutEffect

A Hook like useEffect() that runs earlier in the render lifecycle, often used when layout measurement must happen before paint.

useRef

A Hook that stores a mutable value across renders and is often used to access DOM elements directly.

useMemo

A Hook that memoizes the result of an expensive calculation so it is not recomputed unnecessarily.

useCallback

A Hook that memoizes a function reference to avoid unnecessary re-creations on each render.

useContext

A Hook that reads shared values from React Context without passing props through many layers manually.

Context API

A built-in way to share data like theme or auth state across multiple components without heavy prop drilling.

Prop Drilling

The repeated passing of props through intermediate components that do not need the data themselves.

Controlled Component

A form element whose value is controlled by React state, making the UI the single source of truth.

Uncontrolled Component

A form element that manages its own value in the DOM and is often accessed using refs.

Event Handling

React handles browser events with props such as onClick and onChange, usually pointing to component functions.

Lifting State Up

Moving shared state to a common parent so sibling components can read and update the same source of truth.

Parent-Child Communication

Parents pass data down with props, and children communicate up through callback functions provided by the parent.

Key

A stable identifier used by React to track items in a list and reconcile them correctly during updates.

Conditional Rendering

Showing different UI output depending on state, props, permissions, loading state, or other conditions.

Lifecycle

The high-level phases of a component such as mount, update, and unmount, now usually reasoned about through Hooks.

Custom Hook

A reusable function that combines React Hooks and shared logic into a clean abstraction.

Memoization

A performance technique used to reuse previous values or function references when inputs have not changed.

React.memo

A helper that prevents a component from re-rendering when its props have not changed.

Performance Bottleneck

An area where rendering, data fetching, or computation slows down the app and usually needs measurement before optimization.

Server State

Data that lives on the backend and is fetched, cached, and synchronized separately from local UI state.

Redux / Zustand

Examples of state management libraries used when app-wide state or update coordination grows beyond simple local state and Context.

Anti-pattern

A code habit that makes a React app harder to maintain, such as giant components, unnecessary prop drilling, or premature memoization.

Code Splitting

A technique to load JavaScript in smaller chunks so users do not download the whole app upfront.

Lazy Loading

Loading components or modules only when needed, often improving startup performance.

Unit Testing

Testing a small isolated piece of behavior, such as a component branch, helper, or hook.

Integration Testing

Testing how multiple pieces work together, such as a form, validation flow, and submission result.

Reusable Component

A component with a clear API, minimal assumptions, and behavior that can be reused across screens or features.
