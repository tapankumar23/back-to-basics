PATTERN 07

Backtracking

Exhaustive search over combinatorial space with pruning. Choose → Explore → Unchoose. Three variants: subsets, permutations, constraint satisfaction.

Core Concept

Backtracking explores a decision tree. At each node you choose a candidate, explore deeper, then unchoose (backtrack) to try the next option. Pruning cuts off branches that can never lead to a valid solution.

⊆
Subsets
Include / Exclude each element. 2ⁿ total paths.
π
Permutations
Arrange all elements. n! total paths.
♛
Constraints
Satisfaction: prune early on constraint violation.

Java Templates

// ===== BACKTRACKING SKELETON =====
void backtrack(int[] candidates, int start, int target,
               List<Integer> current, List<List<Integer>> result) {
    if (target == 0) { result.add(new ArrayList<>(current)); return; }
    for (int i = start; i < candidates.length; i++) {
        if (candidates[i] > target) break; // pruning (requires sorted input)
        // Skip duplicates at the same level (for Combination Sum II)
        if (i > start && candidates[i] == candidates[i-1]) continue;
        current.add(candidates[i]);
        backtrack(candidates, i + 1, target - candidates[i], current, result);
        current.remove(current.size() - 1);
    }
}

// ===== N-QUEENS =====
List<List<String>> solveNQueens(int n) {
    List<List<String>> result = new ArrayList<>();
    char[][] board = new char[n][n];
    for (char[] row : board) Arrays.fill(row, '.');
    nqueens(board, 0, result);
    return result;
}
void nqueens(char[][] board, int row, List<List<String>> result) {
    if (row == board.length) {
        List<String> solution = new ArrayList<>();
        for (char[] r : board) solution.add(new String(r));
        result.add(solution);
        return;
    }
    for (int col = 0; col < board.length; col++) {
        if (isValid(board, row, col)) {
            board[row][col] = 'Q';
            nqueens(board, row + 1, result);
            board[row][col] = '.';
        }
    }
}

// ===== SUDOKU SOLVER =====
boolean solveSudoku(char[][] board) {
    for (int r = 0; r < 9; r++) {
        for (int c = 0; c < 9; c++) {
            if (board[r][c] != '.') continue;
            for (char d = '1'; d <= '9'; d++) {
                if (isValidPlacement(board, r, c, d)) {
                    board[r][c] = d;
                    if (solveSudoku(board)) return true;
                    board[r][c] = '.';
                }
            }
            return false; // no digit worked
        }
    }
    return true; // all cells filled
}

Staff Angles

Scale: N-Queens for N=1000

N-Queens for N=1000 via exhaustive backtracking is infeasible — the search space is astronomically large. How do you find one valid solution quickly?

→ Constructive O(n) algorithms exist for N-Queens (place queens using modular arithmetic patterns). Constraint programming solvers (MiniZat, Gecode) use arc-consistency propagation. Las Vegas algorithms randomly restart from promising partial states. For combinatorial optimization at scale, branch-and-bound with good bounding functions prunes orders of magnitude of the tree.

Parallel: Distributing Sudoku Solving

Backtracking is naturally parallelizable at the top levels of the tree. How would you distribute Sudoku solving across 100 workers?

→ Enumerate all valid partial placements for the first 2–3 rows, distribute each subtree to a worker. Work-stealing: idle workers steal top-level nodes from busy workers' queues. Speculative execution of the most-constrained branches first (fail-fast heuristic). Branch-and-bound with shared upper bounds lets workers prune based on each other's findings. Store intermediate results in shared memory to avoid redundant exploration.
