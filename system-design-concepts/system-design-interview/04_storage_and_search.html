Chapters 8–11, 15 — URL Shortener, Web Crawler, Autocomplete, Google Drive

4.1 — URL Shortener Base62 encoding, hash collision, 301 vs 302 redirect
Base62 encoding (a-z, A-Z, 0-9) of a 7-char code gives 62^7 ≈ 3.5 trillion unique URLs. 301 = permanent redirect (browser caches, no future hit to shortener); 302 = temporary redirect (every request hits the shortener server, enabling analytics).
4.2 — Web Crawler BFS traversal, bloom filter dedup, politeness delay
Politeness: wait 1s+ between requests to the same domain to avoid overloading hosts. Bloom filters provide O(1) duplicate URL detection with a tiny false-positive rate and no per-URL storage overhead.
4.3 — Trie Autocomplete Prefix traversal, Top-K results, cache hit
Trie traversal for prefix search is O(p) where p = prefix length. Top-K results are precomputed per node — avoiding full subtree scans. A cache layer returns popular prefix results in O(1), bypassing trie traversal entirely.
4.4 — Google Drive / Cloud Storage Block splitting, deduplication, delta sync, version history
Block-level deduplication: if two users upload the same file, only one copy is stored — a content hash identifies identical blocks. Delta sync uploads only changed blocks, reducing bandwidth by orders of magnitude on large files.
