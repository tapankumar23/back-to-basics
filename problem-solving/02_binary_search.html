PATTERN 02

Binary Search

Input is sorted, OR you can define a monotonic predicate f(x) where the answer is the boundary between false→true. O(log n) per search.

Core Concept

The key insight: you don't need a sorted array — you need a monotonic predicate. If you can answer "is mid a valid answer?" in O(P), the whole search is O(P · log R).

Classic — Find Value

Array is sorted. Shrink [lo, hi] toward target.

lo=0, hi=n-1
while lo <= hi:
  mid = lo + (hi-lo)/2
  if arr[mid] == target → found
  if arr[mid] < target → lo=mid+1
  else → hi=mid-1
Search on Answer

lo/hi are answer-space bounds. Predicate is canAchieve(mid).

lo=minFeasible, hi=maxFeasible
while lo < hi:
  mid = lo + (hi-lo)/2
  if canAchieve(mid) → hi=mid
  else → lo=mid+1
return lo

Java Code Template

// ===== CLASSIC BINARY SEARCH =====
public int binarySearch(int[] arr, int target) {
    int lo = 0, hi = arr.length - 1;
    while (lo <= hi) {
        int mid = lo + (hi - lo) / 2; // avoid overflow
        if (arr[mid] == target) return mid;
        else if (arr[mid] < target) lo = mid + 1;
        else hi = mid - 1;
    }
    return -1;
}

// ===== LOWER BOUND (first index >= target) =====
public int lowerBound(int[] arr, int target) {
    int lo = 0, hi = arr.length; // open interval [0, n)
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (arr[mid] < target) lo = mid + 1;
        else hi = mid;
    }
    return lo;
}

// ===== UPPER BOUND (first index > target) =====
public int upperBound(int[] arr, int target) {
    int lo = 0, hi = arr.length;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (arr[mid] <= target) lo = mid + 1;
        else hi = mid;
    }
    return lo;
}

// ===== SEARCH ON ANSWER TEMPLATE =====
public int searchOnAnswer(int[] arr, int constraints) {
    int lo = minFeasible(arr);   // smallest possible answer
    int hi = maxFeasible(arr);   // largest possible answer
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (canAchieve(arr, mid, constraints)) hi = mid; // find minimum feasible
        else lo = mid + 1;
    }
    return lo;
}

Staff Angles

Scale: Binary Search on Disk

Binary search assumes the array fits in RAM. How do you binary search a 5TB sorted file on disk?

→ B-tree index pages (O(log_B n) I/Os instead of O(log n)), SSTables in LSM trees use bloom filters + sparse index to skip most disk reads, external merge sort produces sorted runs you can index. Each I/O reads a full 4KB page, so binary search with aligned block-granularity jumps. For range scans: prefix-compressed sorted string tables (like LevelDB blocks).

Approximation: Beyond O(n log R)

The canShip predicate is O(n), giving O(n log R) total. At 10B items is this optimal, or is there a better approach?

→ Prefix sums precomputed reduce each predicate call to O(log n) binary search on prefix array → overall O(log n · log R). For distributed systems: distributed binary search on partitioned sorted arrays across shards — each shard answers locally, coordinator aggregates. For approximation: sample k random items, estimate distribution, skip most of the search space entirely (probabilistic O(1) amortized for well-distributed inputs).
