// ─── XOR Tricks ───────────────────────────────────────────
public int singleNumber(int[] nums) {
    int r = 0;
    for (int n : nums) r ^= n;
    return r;  // duplicates cancel (a^a=0), single remains
}

// Brian Kernighan: n & (n-1) clears lowest set bit each iteration
public int hammingWeight(int n) {
    int c = 0;
    while (n != 0) { n &= (n - 1); c++; }
    return c;
}

public boolean isPowerOfTwo(int n) {
    return n > 0 && (n & (n - 1)) == 0;
}

// DP: dp[i] = dp[i>>1] + (i&1)  — counting bits for 0..n in O(n)
public int[] countBits(int n) {
    int[] dp = new int[n + 1];
    for (int i = 1; i <= n; i++)
        dp[i] = dp[i >> 1] + (i & 1);
    return dp;
}

// ─── Bitmask DP skeleton (TSP / visit-all-nodes) ──────────
// state = subset as int; bit j set means element j is included
for (int mask = 0; mask < (1 << n); mask++) {
    for (int j = 0; j < n; j++) {
        if ((mask >> j & 1) == 1)
            dp[mask] = Math.min(dp[mask ^ (1 << j)] + cost[j], dp[mask]);
    }
}

// ─── Subsets via bitmask enumeration (LC 78) ──────────────
List<List<Integer>> subsets = new ArrayList<>();
for (int mask = 0; mask < (1 << n); mask++) {
    List<Integer> sub = new ArrayList<>();
    for (int j = 0; j < n; j++)
        if ((mask >> j & 1) == 1) sub.add(nums[j]);
    subsets.add(sub);
}

// ─── Missing Number (LC 268): XOR 0..n with array ─────────
public int missingNumber(int[] nums) {
    int xor = nums.length;
    for (int i = 0; i < nums.length; i++) xor ^= i ^ nums[i];
    return xor;
}

Staff Angles

Q: "Bloom filter false-positive rate and memory: for 1B items with 1% FPR, how many bits?"
Formula: m = -n·ln(FPR) / (ln 2)² ≈ 9.6 bits/item → ~1.2 GB for 1B items. Optimal hash functions: k = m/n · ln 2 ≈ 7. Bit manipulation is the core — each hash maps to a bit position via pos = hash % m, checked/set with a single bit array. False positives arise from hash collisions; false negatives are impossible. Trade-off: more bits → lower FPR but more memory; more hashes → lower FPR but more CPU.
Q: "Your permission system has 64 roles. How do you check if a user has all required roles efficiently?"
Store roles as a long bitmask (64-bit). Check: user.roles & required == required — O(1) per check. Grant role j: roles |= (1L << j). Revoke role j: roles &= ~(1L << j). Toggle: roles ^= (1L << j). This fits a single DB column (BIGINT), cache-friendly, no JOINs. Scale: shard by user_id; replicate read-heavy permission checks.
