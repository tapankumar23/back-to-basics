PATTERN 18

Union-Find (DSU)

Disjoint Set Union — O(α(n)) dynamic connectivity with path compression + union by rank

Core Concept

Core Idea

DSU maintains disjoint sets (components). Two core operations: find(x) returns the root representative of x's component; union(a,b) merges two components.

Without optimizations: O(n) per op. With path compression + union by rank: amortized O(α(n)) ≈ O(1), where α is the inverse Ackermann function.

Path Compression + Union by Rank

Path compression: on every find, set parent[x] = root directly — flattens the path.

Union by rank: always attach the smaller-rank tree under the larger-rank root — bounds tree height at O(log n).

When DSU beats BFS/DFS

Use DSU when:
  • Dynamic edge additions (online algorithm)
  • Only connectivity needed, not the path
  • Many repeated connectivity queries
Use BFS/DFS when:
  • Need the actual path between nodes
  • Weighted shortest path (Dijkstra/BF)
  • Directed graphs with SCC (Tarjan/Kosaraju)
BFS/DFS: O(V+E) per query, store full graph
DSU: O(α(n)) per query after O(E) build

Java Code Template

// ===== UNION-FIND (DSU) — Path Compression + Union by Rank =====
class DSU {
    private int[] parent, rank;
    private int components;

    public DSU(int n) {
        parent = new int[n];
        rank   = new int[n];
        components = n;
        for (int i = 0; i < n; i++) parent[i] = i;
    }

    public int find(int x) {
        if (parent[x] != x) parent[x] = find(parent[x]); // path compression
        return parent[x];
    }

    public boolean union(int a, int b) {
        int ra = find(a), rb = find(b);
        if (ra == rb) return false; // already same component
        if (rank[ra] < rank[rb]) { int t = ra; ra = rb; rb = t; }
        parent[rb] = ra;
        if (rank[ra] == rank[rb]) rank[ra]++;
        components--;
        return true;
    }

    public boolean connected(int a, int b) { return find(a) == find(b); }
    public int getComponents() { return components; }
}

// ===== WEIGHTED DSU (for rank/ratio problems) =====
class WeightedDSU {
    int[] parent;
    double[] weight; // weight[x] = weight(x) / weight(parent[x])

    public WeightedDSU(int n) {
        parent = new int[n];
        weight = new double[n];
        for (int i = 0; i < n; i++) { parent[i] = i; weight[i] = 1.0; }
    }

    public int find(int x) {
        if (parent[x] == x) return x;
        int root = find(parent[x]);
        weight[x] *= weight[parent[x]]; // accumulate weight on path compression
        parent[x] = root;
        return root;
    }

    public boolean union(int a, int b, double ratio) { // ratio = val(a) / val(b)
        int ra = find(a), rb = find(b);
        if (ra == rb) return false;
        parent[ra] = rb;
        weight[ra] = ratio * weight[b] / weight[a];
        return true;
    }

    public double query(int a, int b) {
        if (find(a) != find(b)) return -1.0; // not connected
        return weight[a] / weight[b];
    }
}

Canonical Problems

LC 547
Number of Provinces
Direct DSU application; count remaining components after all union ops. Also solvable with BFS but DSU is cleaner and faster.
LC 684
Redundant Connection
Find the first edge whose both endpoints already share a root. The redundant edge forms the cycle — return it immediately.
LC 305
Number of Islands II
Online: islands added dynamically one at a time. After each add, union with adjacent land cells and query component count. DSU shines here; BFS would be O(k·mn).
LC 721
Accounts Merge
DSU on email indices (not strings). Key insight: map each email to an integer ID, union IDs sharing an account, then group emails by root component.
LC 947
Most Stones Removed (Same Row/Col)
Union stones sharing same row or column. Answer = n − (number of components). Elegant: a component can be reduced to 1 stone.
LC 1202
Smallest String With Swaps
DSU on indices: union swappable pairs. Within each component, collect characters, sort, and place back in sorted index order.
LC 399
Evaluate Division
Weighted DSU where edge weight = ratio a/b. Path compression accumulates weights multiplicatively. Query = weight[a]/weight[b] when same root.
LC 827
Making a Large Island
Label each component with DSU and record sizes. For each 0-cell, sum distinct adjacent component sizes + 1. Two-pass O(mn) using DSU labels.

Staff Angles

Distributed DSU

You have a social graph with 1B nodes partitioned across 50 machines. Users connect dynamically. How do you maintain connected components? Discuss local DSU per shard + periodic global merge rounds vs. consistent-hash-based routing. What's the consistency model?

→ Local DSU per shard handles intra-shard edges in O(α(n)); cross-shard edges require a two-phase protocol — route both endpoints to a coordinator or use consistent hashing so all nodes in a component land on the same shard (but rebalancing is expensive). Periodic global merge (MapReduce-style): each round ships root IDs across shards, takes O(E/shards) messages per round, converges in O(log n) rounds. Consistency model: eventual — queries during a merge window may return stale component info. For strong consistency, serialize cross-shard unions through a distributed lock or use a CRDTs-based union-find where merges are idempotent and commutative.

Persistent / Rollback DSU

You need to support undo operations (rollback last k unions) — for example, in an offline algorithm (like link-cut trees or offline LCT) that processes queries in a different order. Path compression breaks rollback. How do you handle this?

→ Drop path compression; keep union by rank only. Tree height stays O(log n), so find is O(log n) — acceptable. Rollback: maintain a stack of (node, oldParent, oldRank) tuples. Each union pushes 1–2 entries. Rollback k unions: pop and restore exactly k times, O(k log n) total. This technique enables offline divide-and-conquer on dynamic connectivity: split timeline into segments, process with rollback DSU in O(E log E log n). For fully persistent DSU with O(log n) per op, use a persistent array (path-copying) instead — but that's O(n log n) space per version.
