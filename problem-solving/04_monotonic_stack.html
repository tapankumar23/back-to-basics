Monotonic Stack

Use when you need the next/previous greater or smaller element, histogram areas, or temperature spans. Maintain a stack monotonically increasing or decreasing — pop and process when the invariant is violated.

Signal — "next/previous greater/smaller"
Time O(n) amortized
Space O(n) stack
Pop trigger — invariant violated

Core Concepts

A monotonic stack keeps elements in a sorted order. When the current element violates the order, we pop and process — each element is pushed and popped at most once, giving O(n) total work.

📈

Increasing Stack

Pop when arr[i] is smaller than top. Used for next smaller element, largest rectangle.

📉

Decreasing Stack

Pop when arr[i] is larger than top. Used for next greater element, daily temperatures.

🏗️

Store Indices

Almost always store indices, not values. Lets you compute widths, spans, distances.

The Pop Moment

When you pop index j because arr[i] triggered it: arr[i] is the next greater element for j. The new stack top is the previous greater element. One pop gives you both answers.

Sentinel Trick

Append a sentinel value (0 or -infinity) to flush the stack at the end. Avoids a separate post-loop cleanup pass for histogram and rectangle problems.

MonotonicStack.java 
// NEXT GREATER ELEMENT — O(n) decreasing monotonic stack
public int[] nextGreaterElement(int[] nums) {
    int n = nums.length; int[] result = new int[n]; Arrays.fill(result, -1);
    ArrayDeque<Integer> stack = new ArrayDeque<>();
    for (int i = 0; i < n; i++) {
        while (!stack.isEmpty() && nums[stack.peek()] < nums[i])
            result[stack.pop()] = nums[i];
        stack.push(i);
    }
    return result;
}

// LARGEST RECTANGLE IN HISTOGRAM — sentinel trick
public int largestRectangleArea(int[] heights) {
    int n = heights.length, maxArea = 0;
    ArrayDeque<Integer> stack = new ArrayDeque<>();
    for (int i = 0; i <= n; i++) {
        int h = (i == n) ? 0 : heights[i];
        while (!stack.isEmpty() && heights[stack.peek()] > h) {
            int height = heights[stack.pop()];
            int width = stack.isEmpty() ? i : i - stack.peek() - 1;
            maxArea = Math.max(maxArea, height * width);
        }
        stack.push(i);
    }
    return maxArea;
}

// TRAPPING RAIN WATER — two-pointer O(1) space
public int trap(int[] height) {
    int left=0, right=height.length-1, water=0, maxL=0, maxR=0;
    while (left < right) {
        if (height[left] <= height[right]) {
            maxL = Math.max(maxL, height[left]);
            water += maxL - height[left++];
        } else {
            maxR = Math.max(maxR, height[right]);
            water += maxR - height[right--];
        }
    }
    return water;
}

Staff Engineer Angles

"Maintain 'previous greater element' for every insert in a stream of 10B stock prices with a 30-day window, bounded memory?"
Use a sliding window monotonic deque (ArrayDeque). Evict elements outside the 30-day window from the front. Each new price evicts smaller prices from the back. Store (price, timestamp) tuples. Memory is bounded to window size O(30). Time bucketing with periodic compaction reduces overhead for burst traffic. Total: O(1) amortized per insert.
"Largest rectangle across a histogram split across 10 shards without merging full data?"
Divide-and-conquer: each shard computes its internal max rectangle independently. Boundary rectangles spanning shards require the minimum height across the boundary region — each shard exposes a range-min segment tree. During merge, binary search on width and query range-min to find cross-shard candidates. Total work: O(n log n) with O(k) inter-shard messages where k is shard count.
