Core Distributed Systems

Chapters 5 & 6 — Consistent Hashing, Key-Value Store, CAP, Gossip, Quorum, Merkle Trees

2.1 — Consistent Hashing Minimal key migration on node join/leave
A node join/leave only moves ~1/N of the keys to the adjacent neighbor — not a full reshuffle. Virtual nodes (vnodes) spread each server across multiple ring positions, giving even load distribution even when server capacities differ.
2.2 — Key-Value Store Internals WAL, MemTable, Bloom Filter, SSTable compaction
WAL (Write-Ahead Log) ensures durability — even a crash cannot lose a committed write. MemTable is in-memory for fast writes; it flushes to an SSTable on disk when full. Bloom filters are probabilistic — a false positive sends you to the SSTable, but a true negative spares the expensive disk read.
2.3 — CAP Theorem Consistency, Availability, Partition Tolerance — pick two
In a distributed system, network partitions are inevitable. When a partition occurs you must choose: reject writes to stay consistent (CP) or accept stale reads to stay available (AP). CA systems (MySQL single-node) exist only without partitions — i.e., on a single machine.
2.4 — Gossip Protocol O(log N) rounds to convergence — no central coordinator
Round: 0
Gossip spreads state in O(log N) rounds. Each infected node randomly picks 2–3 neighbors to infect. No node needs to know the full topology — every node eventually reaches the same state without central coordination.
2.5 — Quorum & Replication W + R > N guarantees strong consistency
W: 3 R: 3
W + R > N guarantees at least one node in the read set saw the latest write. Tuning W and R trades consistency for latency: W=1, R=1 is fastest but eventually consistent; W=N is slowest but ensures every node has the write before acknowledging.
2.6 — Merkle Trees O(log N) diff detection between replicas
Merkle trees let two replicas find their differences in O(log N) comparisons instead of comparing all data. The comparison starts at the root — if root hashes match, the replicas are identical. Divergence is pinpointed by walking down only the mismatched subtree.
