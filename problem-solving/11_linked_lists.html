11 · Linked Lists

Master in-place O(1)-space manipulation using three canonical tools: dummy head, slow/fast pointers, and recursion. Floyd's cycle detection is the signature algorithm — elegant, O(1) space, and the entry point for a dozen interview problems.

Core Concept

Anatomy of a Linked List

Each node holds a value and a next pointer. The last node's next = null. A dummy/sentinel head simplifies edge cases when inserting at the front.

Dummy Head Pattern

Create a fake node before the real head. Return dummy.next at the end. Eliminates null-checks when stitching nodes onto an initially empty result list. Used in: merge two lists, remove nth from end, partition list. 

ListNode dummy = new ListNode(0);
ListNode cur = dummy;
// stitch nodes onto cur.next
cur.next = someNode;
cur = cur.next;
return dummy.next;

Slow / Fast Pointer Pattern

Slow moves 1 step; fast moves 2 steps per iteration. When fast reaches the end, slow is at the middle. If fast ever equals slow (started apart), there is a cycle. Used in: find middle, cycle detection, palindrome check. 

ListNode slow = head, fast = head;
while (fast != null && fast.next != null) {
  slow = slow.next;
  fast = fast.next.next;
}
// slow is now at the middle

Java Implementation

Reverse Linked List (Iterative + Recursive)

// Iterative — O(n) time, O(1) space
public ListNode reverseList(ListNode head) {
    ListNode p = null, c = head;
    while (c != null) {
        ListNode n = c.next;   // 1. save next
        c.next = p;             // 2. reverse link
        p = c;                  // 3. advance prev
        c = n;                  // 4. advance cur
    }
    return p;                  // new head
}

// Recursive — O(n) time, O(n) stack space
public ListNode reverseListRec(ListNode head) {
    if (head == null || head.next == null) return head;
    ListNode newHead = reverseListRec(head.next);
    head.next.next = head;   // wire successor back to us
    head.next = null;        // break forward link
    return newHead;
}

Floyd's Cycle Detection — hasCycle & detectCycle

// LC 141 — O(n) time, O(1) space — no HashSet
public boolean hasCycle(ListNode h) {
    ListNode s = h, f = h;
    while (f != null && f.next != null) {
        s = s.next;
        f = f.next.next;
        if (s == f) return true;
    }
    return false;
}

// LC 142 — find the exact cycle-entry node
// Math: dist(head→entry) == dist(meet→entry going forward)
public ListNode detectCycle(ListNode h) {
    ListNode s = h, f = h;
    while (f != null && f.next != null) {
        s = s.next;
        f = f.next.next;
        if (s == f) {
            s = h;                  // reset slow to head
            while (s != f) {      // both advance 1 step
                s = s.next;
                f = f.next;
            }
            return s;            // cycle start node
        }
    }
    return null;
}

Remove Nth Node From End — Single Pass

// LC 19 — dummy head + two pointers (n+1) apart
public ListNode removeNthFromEnd(ListNode head, int n) {
    ListNode dummy = new ListNode(0);
    dummy.next = head;
    ListNode fast = dummy, slow = dummy;
    for (int i = 0; i <= n; i++) fast = fast.next; // n+1 gap
    while (fast != null) {
        slow = slow.next;
        fast = fast.next;
    }
    slow.next = slow.next.next;  // unlink target node
    return dummy.next;
}

Problem Set

    Reverse Linked List Easy
    Iterative (prev/cur/next triple swap) or recursive (recurse to tail, wire back). Iterative is O(1) space; recursive is O(n) stack.
  • 92
    Reverse Linked List II Medium
    Reverse only between positions m and n in a single pass. Anchor the node just before position m with a dummy head to avoid null checks.
  • 141
    Linked List Cycle Easy
    Floyd's slow/fast: if they ever meet, a cycle exists. O(1) space — no HashSet needed at all.
  • 142
    Linked List Cycle II Medium
    After phase-1 meeting point, reset slow to head; advance both 1 step per iteration — they meet exactly at the cycle entry node.
  • 876
    Middle of Linked List Easy
    Slow/fast: when fast reaches null, slow is at the middle. For even length lists this returns the second middle node.
  • 21
    Merge Two Sorted Lists Easy
    Dummy head plus pointer stitching: always attach the smaller current node, then advance that list's pointer. Append remaining tail at end.
  • 23
    Merge K Sorted Lists Hard
    Min-heap of (value, listIndex) for O(N log k); alternatively divide-and-conquer merging pairs achieves the same complexity with no heap overhead.
  • 19
    Remove Nth From End Medium
    Gap of n+1 between fast and slow (both starting from dummy). When fast becomes null, slow.next is the target — unlink with slow.next = slow.next.next.
  • 160
    Intersection of Two Lists Easy
    After reaching the end of one list, switch that pointer to the other list's head. Both pointers walk equal total distance and meet at the intersection or null.
  • 234
    Palindrome Linked List Easy
    Find middle with slow/fast, reverse the second half in-place, compare both halves. Optionally restore the list after comparison.
  • 138
    Copy List with Random Pointer Medium
    HashMap old-to-new in two passes; or interleave cloned nodes (A then A' then B then B'), wire randoms, then split — achieves O(1) extra space.
  • 146
    LRU Cache Medium
    Doubly linked list (most-recent at tail) plus HashMap of key-to-node. O(1) get/put — the classic design interview sub-problem.

Staff / System-Design Angles

"LRU cache — 100 million entries at 64 bytes/node = 6 GB on-heap. How do you reduce the memory footprint?"
Off-heap storage: use sun.misc.Unsafe or libraries like Chronicle Map / EHCache to allocate node memory outside the JVM heap, eliminating GC pressure entirely.

Arena allocators: pre-allocate a fixed slab and hand out fixed-size slots. Eliminates per-object header overhead (~16 bytes on JVM) and improves cache locality significantly.

Struct-of-arrays (SoA): store keys[], values[], prev[], next[] as parallel primitive arrays instead of pointer-rich objects. Reduces per-entry overhead from ~64 B to ~24 B and is SIMD-friendly.

Compression + tiering: apply varint encoding for keys; use value compression (Snappy) for large payloads. Hot set lives in DRAM, warm set spills to SSD (RocksDB or similar).
"90% of our LRU latency is lock contention. How do you fix it?"
Sharded LRU: partition by key.hashCode() % N into N independent LRU instances, each with its own lock. N=64 shards typically cuts contention by ~64x in practice.

Caffeine / W-TinyLFU: Caffeine uses a lock-free ring buffer for access recording; a background thread drains and reorders the eviction policy. Sub-microsecond get latency with near-optimal hit rate.

CAS / lock-free operations: AtomicReference plus compare-and-swap for pointer updates. Works well when write contention is moderate; degrades under very high write load.

Async access recording: reads do a best-effort async bump via a bounded queue. A missed bump degrades eviction quality slightly but never breaks correctness — a practical trade-off at very high throughput.
