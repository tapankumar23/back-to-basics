01 — Hash Index
Append-Only Log + In-Memory Hashmap
Bitcask-style storage: every write appends to a segment file. A hash map in memory maps each key to its latest byte offset.
Segment Log
In-Memory Hashmap (key to offset)
Compaction: duplicate keys removed; segments merged into one compacted file. Reads still served from old segments until merge completes.
02 — LSM-Tree + SSTables

Writes go to an in-memory MemTable. When it fills, it is flushed to an immutable SSTable on disk. Background compaction merges SSTables level by level.

03 — B-Trees
Update-in-Place with WAL Safety
B-trees update leaf pages in place. A Write-Ahead Log (WAL) is written first for crash recovery. When a page overflows, it splits.
B-Tree Structure
WAL + Crash Recovery
WAL: every write is first appended to the Write-Ahead Log. On crash, the WAL is replayed to restore the B-tree to a consistent state.

05 — OLTP vs OLAP
Transactional vs Analytical Workloads
OLTP handles many small reads/writes for users. OLAP runs large aggregations over historical data for analysts.
Currently: OLTP mode
06 — Column-Oriented Storage
Store Columns, Not Rows
Column stores read only the columns needed for a query, enabling much better I/O efficiency for analytics. Run-length encoding compresses repeated values dramatically.
Row vs Column Store
Run-Length Encoding
Compression ratio: 6:3 = 2x. Sorted columns compress even better. Allows vectorized CPU operations on compressed data.

10 — Modes of Dataflow
Databases, Services, Message Passing
Encoded data flows between processes in three major modes. Each has different compatibility requirements, failure modes, and deployment patterns.
Dataflow Through Databases ►
Writer encodes, bytes rest at rest, future reader decodes. Different versions of code may coexist.
Key insight: when you deploy a new version of your app, it may read records written by the old version and vice-versa. Use schema evolution carefully — add optional fields, never remove required fields.
Dataflow Through Services (REST vs RPC) ►
REST uses HTTP verbs on resources. RPC looks like local calls but hides network failures.
RPC leaky abstraction: local calls cannot timeout, cannot fail partway, and cannot have ambiguous results. Network calls can. gRPC, Thrift, Avro RPC all try to hide this — be careful.
Message-Passing Dataflow ►
Producer to broker to consumer. Decouples sender and receiver in time and space.
Advantages: buffer if receiver is slow, redeliver on crash, fan-out to multiple consumers, decouple sender from receiver. Actor model uses single-threaded mailboxes for concurrency without locks.
