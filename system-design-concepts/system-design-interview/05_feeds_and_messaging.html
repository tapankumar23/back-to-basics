Feeds & Messaging

Chapters 10–12 — News Feed, Notifications, Chat System

5.1 — News Feed: Fan-out on Write (Push) Pre-computed feeds written to every follower's cache
Fan-out on write (push) pre-computes each user's feed. Reads are fast (O(1)) but writes for celebrities with millions of followers create hot spots — a single post can trigger 500K+ cache writes simultaneously.
5.2 — News Feed: Fan-out on Read (Pull) Feed computed at read time by querying all followees
Fan-out on read (pull) computes the feed at read time. Always fresh data, but latency increases with followee count (~500ms for many followees). Heavy for popular users who follow many people.
5.3 — Hybrid Fan-out Push for regular users, pull for celebrities (>10K followers)
REGULAR USER
Hybrid approach: push for regular users (fast reads, pre-written feeds), pull for celebrities (avoid hot spots). The threshold is typically ~10K followers. On feed open, both pre-written posts and on-demand celebrity posts are merged.
5.4 — Notification System Multi-channel fan-out with retry and exponential backoff
Notifications are fire-and-forget with retry. Exponential backoff prevents overloading a struggling downstream provider. Idempotency keys prevent duplicate notifications on retries.
5.5 — Chat System (WebSocket) Persistent bidirectional connections via WebSocket
WebSocket provides persistent bidirectional connections — no polling overhead. Chat servers are stateful (they know which users are connected). A message queue decouples the two chat servers so they don't need direct connections.
